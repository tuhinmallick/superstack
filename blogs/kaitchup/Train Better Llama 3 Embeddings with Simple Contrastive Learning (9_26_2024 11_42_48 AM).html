The method uses a masked next-token prediction (MNTP) objective for training. MNTP adapts the embeddings of an LLM for the task of encoding text. However, embeddings extracted from LLMs and only trained with MNTP still significantly underperform embedding models trained from scratch.

To improve the performance of an embedding model, contrastive learning is usually employed. It only requires in-domain text, for the unsupervised variant, or a dataset annotated for textual entailment for the supervised version.

Simple Contrastive Learning for LLMs

In this article, I use the method presented in this paper:

SimCSE: Simple Contrastive Learning of Sentence Embeddings

The idea behind SimCSE is very simple.

For a given input sequence, it undergoes two separate passes through the embedding model, each time using independently sampled dropout masks. This process yields two distinct representations of the same sequence.

Dropout

Dropout is a regularization technique employs to train large neural network. In SimCSE, you can see it as a technique randomly removing some token, or altering its embeddings, from a training sequence. It makes the model less confident in its modelization, and thus learns slower and better.

The SimCSE training objective is to improve the similarity between these two representations while reducing their similarity to representations of other sequences within the same batch.

An important advantage of this method is that it can be unsupervised as it does not rely on paired data and can be used with any set of sequences. There is also a supervised variant which, on the other hand, requires paired data.

As shown by the LLM2VEC paper, unsupervised SimCSE can already achieve a substantial improvement for embedding models only trained with MNTP objective:

Training Embedding Models with SimCSE

LLM2VEC also provides the code to train embedding models with SimCSE. In this section, we will see how to use it. I will train an embedding model extracted from Llama 3 which has already been trained with the MNTP objective. To reproduce this setup, check this article:

Note: When I wrote this article ↑ using LLM2VEC, the code for SimCSE was not released yet.

The code for SimCSE only supports three architectures: Mistral, Llama, and Gemma (and probably NOT Gemma 2).

Setup

For SimCSE training, we need to install the following:

pip install --upgrade llm2vec transformers

It supports FlashAttention which is only compatible with recent GPUs. If your GPU is compatible (e.g., RTX 3xxx/4xxx or more recent), install FlashAttention to reduce the memory consumption:

pip install flash-attn --no-build-isolation

Since we will use a script from the LLM2VEC repository, we need to download the code:

git clone https://github.com/McGill-NLP/llm2vec.git

Then, it remains to register in your environment your Hugging Face access token in case you use Llama 3 8B for the first time:

huggingface-cli login

SimCSE with LLM2VEC for Llama 3

First, we need training data. For this tutorial, since I’m not targeting any particular domain, I chose the same dataset used by the authors of LLM2VEC: a Wikipedia extract. For real-world tasks, you should use datasets as close as possible to your target tasks. It only needs to be sequences of tokens, i.e., raw text would work.

wget https://huggingface.co/datasets/princeton-nlp/datasets-for-simcse/resolve/main/wiki1m_for_simcse.txt

Then, we define the configuration for this training. I set the training arguments as follows:

{
    "model_name_or_path": "meta-llama/Meta-Llama-3-8B",
    "peft_model_name_or_path": "./Meta-Llama-3-8B-llm2vec-MNTP-Emb",
    "simcse_dropout": 0.3,
    "bidirectional": true,
    "pooling_mode": "mean",
    "dataset_name": "Wiki1M",
    "dataset_file_path": "wiki1m_for_simcse.txt",
    "remove_unused_columns": false,
    "learning_rate": 3e-5,
    "do_train": true,
    "loss_scale": 20,
    "per_device_train_batch_size": 1,
    "gradient_accumulation_steps": 32,
    "disable_tqdm": false,
    "save_steps": 200,
    "stop_after_n_steps": 1000,
    "max_seq_length": 128,
    "lora_r": 16,
    "gradient_checkpointing": true,
    "torch_dtype": "bfloat16",
    "attn_implementation": "flash_attention_2",
    "output_dir": "./Meta-Llama-3-8B-llm2vec-SimCSE-Emb"
}

There are only a few important differences with what the authors of LLM2VEC did in their paper.

First, I used the Llama 3 8B base, not the instruct version. As shown by recent work, “A Tale of Trust and Accuracy: Base vs. Instruct LLMs in RAG Systems”, using the base version of LLMs for RAG instead of their instruct version can yield better results. I assume this might also be true for an embedding model that we intend to use for RAG and extracted from base LLMs.

In the configuration above, “peft_model_name_or_path” is an adapter for the embedding model trained with the MNTP objective (training code also provided in the notebook). For SimCSE training, this adapter is merged into the base model before training starts. This merge is automatically performed by the script.

If your machine doesn’t support FlashAttention, remove “attn_implementation": "flash_attention_2".

To run the training, use this script:

python llm2vec/experiments/run_simcse.py SimCSE_config_Meta-Llama-3-8B.json

It took a few hours with the L4 instance of Google Colab.

The output is an adapter since the training script uses LoRA.

To make the final embedding model trained with MNTP and SimCSE, merge first the MNTP adapter and then the SimCSE adapter into the base model.

Conclusion

We can easily improve the embedding models extracted from LLMs through contrastive learning.

According to the authors of LLM2VEC, SimCSE training might be enough to make embeddings extracted from LLMs as good as state-of-the-art embedding models trained from scratch.

The only downside that I can see with this approach is that it produces embedding models that are much larger than regular embedding models, i.e., they consume much more memory. This is mainly because embeddings from LLMs have a much larger vocabulary and dimensions than regular embedding models.
