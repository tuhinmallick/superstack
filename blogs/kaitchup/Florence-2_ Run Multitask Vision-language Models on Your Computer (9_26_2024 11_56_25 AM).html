Florence-2 by Microsoft are state-of-the-art vision-language models (VLMs) designed to tackle an extensive array of vision and vision-language tasks using a prompt-based approach. The models are remarkably small and Florence-2 is better than other VLMs 100x larger.

Two versions are available, 230M and 770M. The 770M version is small enough to run on a 4 GB GPU.

The Neural Architecture of Florence-2

Florence-2 is described in this paper:

Florence-2: Advancing a Unified Representation for a Variety of Vision Tasks

Florence-2 is a vision-language model. In other words, it is multimodal. It takes as input an image and prompt describing the task to achieve.

The image is encoded with visual embeddings and the textual prompt with text embeddings using BERT. There is also a third type of embeddings which are the location embeddings. They refer to pixels, or coordinates, from the input image.

The models use a classic encoder-decoder architecture. The decoding part is a language model that generates token after token, including “location tokens” indicating positions on the image. As we will see in the next section, these location tokens can be used to delimit areas on the image. For instance, if the prompt requests the model to find a specific object on the image, the model will generate 4 points that are coordinates that can be used to draw a bounding box on the image. This bounding box contains, according to the model, the specific object requested in the prompt.

Florence-2 is available in two sizes, and each size is available as a base and fine-tuned model (all with an MIT license) :

  • microsoft/Florence-2-base: 230M parameters

  • microsoft/Florence-2-large: 770M parameters

If you are more familiar with large language models (LLMs), which usually exploit billions of parameters, you will find that Florence-2 models are quite small. Most modern consumer GPUs, even low-end GPUs, have more than enough memory to run them.

If we look closer at their architecture, we can see that Florence-2 large contains only 12 encoder layers and 12 decoder layers, while the hidden dimensions are only 1,024. To train these VLMs, Microsoft created a new training dataset, FLD-5B, encompassing a total of 5.4B annotations automatically generated for 126M images.

FLD-5B contains annotations for the following tasks (green boxes in the figure above):

  • OCR

  • Captioning

  • Grounding

  • Object detection

  • Segmentation

All these annotations were generated by specialized models, e.g., OCR annotations were generated by the Azure OCR API.

In a way, Florence-2 is a product of knowledge distillation. Florence-2 are student models trained on data generated by teacher models, potentially larger (we don’t have many details on these models), but specialized in particular tasks. It results in models that are much smaller than the teacher models while approximating their accuracy for each task.

Microsoft masters this kind of approach as we already saw with the Phi models which are small language models trained on data generated by larger models.

Running Florence-2 Locally on Your Computer

Note: To write this section, I borrowed code from the official notebook released by Microsoft with Florence-2. You can find this notebook here.

Requirements

First, we need to install the following:

pip install einops timm flash_attn accelerate

Then, import:

from transformers import AutoProcessor, AutoModelForCausalLM
from PIL import Image, ImageDraw, ImageFont
import random
import numpy as np
import requests
import copy
%matplotlib inline

Loading Florence-2 and Its Multimodal Processor

To load the model, it’s not different from an LLM. In this article, I only experiment with Florence-2 large but if you need a smaller model you can load the base version instead. It would work the same.

model_id = 'microsoft/Florence-2-large'
model = AutoModelForCausalLM.from_pretrained(model_id, trust_remote_code=True, device_map='cuda')

Note: We pass “trust_remote_code=True” since the model requires additional code, provided in the repository, that is automatically executed by Hugging Face Transformers.

We also need a processor to process the multimodal input. It combines a text processor, i.e., a tokenizer, and an image processor. 

processor = AutoProcessor.from_pretrained(model_id, trust_remote_code=True)

Inference with Florence-2

Now that everything is set up, we can write the code for inference.

The input of Florence-2 is multimodal: a prompt and an image. We need to process them:

inputs = processor(text=task_prompt, images=image, return_tensors="pt")

It extracts the pixel values and tokenizes the prompt into token IDs. The next step is a call to generate:

    generated_ids = model.generate(
      input_ids=inputs["input_ids"].cuda(),
      pixel_values=inputs["pixel_values"].cuda(),
      max_new_tokens=1024,
      early_stopping=False,
      do_sample=False,
      num_beams=3,
    )

It’s not very different from what we would do with LLMs. The main difference is that we pass pixel values for inference in addition to the token IDs.

Finally, it remains to post-process the output: detokenize and, depending on the task, extract the location tokens.

generated_text = processor.batch_decode(generated_ids, skip_special_tokens=False)[0]
parsed_answer = processor.post_process_generation(
        generated_text,
        task=task_prompt,
        image_size=(image.width, image.height)
)

Demonstration with Examples

For example, I used two images for which we will try the following tasks:

  • captioning

  • detailed captioning

  • object detection

  • OCR

Note: You will find more tasks in the notebook.

The first image is rather simple: A bee and flowers.

The second image is much more challenging for Florence-2: Times Square at night.

With this second image, since it contains numerous billboards, we can also try the OCR capabilities of Florence-2.

We first need to load the image with Python:

url1 = "https://huggingface.co/datasets/huggingface/documentation-images/resolve/main/transformers/tasks/bee.JPG?download=true"
image1 = Image.open(requests.get(url1, stream=True).raw)


url2 = "https://images.unsplash.com/photo-1601751664209-be452817a8ce?q=80&w=2574&auto=format&fit=crop&ixlib=rb-4.0.3&ixid=M3wxMjA3fDB8MHxwaG90by1wYWdlfHx8fGVufDB8fHx8fA%3D%3D"
image2 = Image.open(requests.get(url2, stream=True).raw)

Then, we can query the model with our different tasks. Florence-2 has been trained to understand tasks through tags. For instance, the prompt must contain the tag “<CAPTION>” for captioning.

Captioning

task_prompt = '<CAPTION>'
run_example(task_prompt,image1)

It generates:

{'<CAPTION>': '\nA pink flower with a bee on it.\n'}

and for the second image:

{'<CAPTION>': '\nA crowd of people walking down a busy city street at night.\n'}

It’s very good and short.

Detailed captioning

task_prompt = '<DETAILED_CAPTION>'
run_example(task_prompt, image1)

It generates:

{'<DETAILED_CAPTION>': '\nThe image shows a bee perched on a pink cosmos flower surrounded by other pink and red flowers and green leaves in the background, which is slightly blurred.\n'}

and for the second image:

{'<DETAILED_CAPTION>': '\nThe image shows a bustling Times Square in New York City at night. There are many people walking around, some carrying bags, and vehicles driving on the road. The buildings in the background are illuminated with bright lights and boards with text, and a clock tower stands tall in the distance. The sky is dark, adding to the atmosphere of the bustling city.\n'}

It’s impressive. The model could recognize many details of the second image and knows it’s Times Square.

Object Detection with Florence-2

task_prompt = '<OD>'
results = run_example(task_prompt,image1)
plot_bbox(image1, results['<OD>'])

and

Even if the second image is crowded with objects, it could spot many of them. Most of the billboards were detected and correctly separated.

OCR Using Florence-2

I also tried OCR, i.e., detecting areas with text on the image and generating the corresponding text.

task_prompt = '<OCR>'
run_example(task_prompt,image2)

I tried with the first and the model correctly returned nothing since there isn’t any text on this image.

For the second image, it returns text that appears on the image but it’s not very well segmented. The model also has some issues with the recognition of spaces. It could probably benefit from some fine-tuning on more OCR tasks:

{'<OCR>': "\nBRANDEECITES2019CALLSTHEAMERICA'S NEW YORKtheWonderfulMARRIOTTcompany.MARRIOTMARQUISFEEL THENextLOVELOVESTAYCONNECTEDWITHWITHMUSICMUSICTEXT SAVETO 414.vhvashavethemusic.comvhavishavethemusic.com-MoblleRideWave!WE FRIENDS\n"}

I didn’t try the fine-tuned version also released by Microsoft. They should perform better for most of these tasks.

Conclusion

Florence-2 is very impressive. Despite its small size, Florence-2 large is very capable and useful for many different tasks. The possible applications for this kind of multitask VLM are numerous.

In this article, we focused on exploring the base model. In an upcoming article, we will see how to fine-tune it to create a multimodal chat model.
