Alibaba’s Qwen2-VL are vision language models now available as 2B and 7B parameter models. They are generative language models that support multimodal inputs. You can provide Qwen2-VL, along with text, a single image, multiple images, or a 20-minute video!

The models demonstrate impressive performance in visual understanding. Like Microsoft’s Florence-2, Qwen2-VL can perform many types of tasks such as OCR, image captioning, question answering, visual grounding, etc.

Qwen2-VL: Architecture and Training

Qwen2-VL’s Architecture

Qwen2-VL is based on Qwen-VL with a few modifications. Qwen-VL was presented in this paper:

Qwen-VL: A Versatile Vision-Language Model for Understanding, Localization, Text Reading, and Beyond

The Qwen-VL architecture integrates three main components:

  • A large language model (LLM), initialized with Qwen2 weights for Qwen2-VL.

  • A visual encoder using the Vision Transformer (ViT) architecture, specifically pre-trained with weights from Openclip’s ViT-bigG to process images by resizing them and splitting them into patches, generating image features for further processing.

  • A position-aware vision-language adapter that compresses the image features using a cross-attention mechanism with trainable query vectors. This mechanism also integrates 2D positional encodings to preserve positional information, reducing the visual feature sequence to a manageable fixed length, which is then processed by the LLM.

Qwen2-VL also exploits a new Multimodal Rotary Position Embedding (M-ROPE). The positional embedding is decomposed into components that capture 1D textual, 2D visual, and 3D video positional information, thereby improving the model's ability to process multimodal data.

Qwen2-VL’s Training

Qwen2-VL was trained in several stages, beginning with a pre-training phase that focuses on optimizing the visual encoder and adapter while keeping the LLM frozen.

This phase uses a large, cleaned dataset of image-text pairs, aiming to minimize the cross-entropy of text tokens. The subsequent multi-task pre-training stage involves training the entire model, including the LLM, on a range of visual-language tasks, using high-quality, fine-grained data, and increasing the input resolution of the visual encoder to reduce information loss.

Finally, the models underwent instruction fine-tuning to improve their dialogue and instruction-following capabilities. During this stage, the visual encoder is frozen, and the LLM and adapter are optimized using a mix of multi-modal and pure text dialogue data.

Prompt Format for Qwen2-VL

The prompt format is simple. Replace {Your Prompt} with your prompt in the following template:

<|im_start|>system\nYou are a helpful assistant.<|im_end|>\n<|im_start|>user\n<|vision_start|><|image_pad|><|vision_end|>{Your Prompt}<|im_end|>\n<|im_start|>assistant\n'

Then, the images/video and the prompt are passed to a method that will encode everything.

In practice, we won’t write this template by ourselves. We will use the tokenizer chat template and provide image/video and text as a JSON object, as in the example below:

messages = [
    {
        "role": "user",
        "content": [
            {
                "type": "image",
                "image": "image.jpeg",
            },
            {"type": "text", "text": "Talk about this image."},
        ],
    }
]

Qwen2-VL GPU Requirements

Qwen2-VL 7B Requirements

I recommend a recent GPU (Ampere generation or more recent) to enable bfloat16 and FlashAttention. Otherwise, running the 7B model on a consumer GPU will be challenging.

The 7B model requires a 24 GB GPU. During my experiments, it consumed 20.6 GB of VRAM.

If you have a 16 GB GPU, you will need to run one of the quantized 4-bit versions provided by Alibaba (GPTQ or AWQ).

Qwen2-VL 2B Requirements

For the 2B version, a 12 GB GPU is enough. The 4-bit version should work on an 8 GB GPU. In my experiments, the GPTQ 4-bit version consumed up to 7.3 GB.

Run Qwen2-VL with Single and Multiple Images

Alibaba provides a library that I recommend to install to facilitate the processing of the multimodal input:

pip install qwen-vl-utils

If you use a quantized version, you will need to install auto-gptq or autoawq, and optimum:

!pip install autoawq auto-gptq optimum

Then, after, install transformers and accelerate. The stable version of Transformers didn’t support Qwen2-VL when I wrote this so I installed it from source:

pip install git+https://github.com/huggingface/transformers accelerate

If you are going to process videos, the “av” package is also required:

pip install av

Let’s see a first example of inference, for a single image.

I used images from this dataset:

  • SakanaAI/JA-Multi-Image-VQA

The image I chose to have my first chat with Qwen2-VL is this one:

First, we import what we will need:

from transformers import Qwen2VLForConditionalGeneration, AutoTokenizer, AutoProcessor
from qwen_vl_utils import process_vision_info
import torch

Then:

model_name = "Qwen/Qwen2-VL-2B-Instruct"

model = Qwen2VLForConditionalGeneration.from_pretrained(
    model_name, torch_dtype=torch.bfloat16, attn_implementation="flash_attention_2", device_map="auto"
)

processor = AutoProcessor.from_pretrained(model_name)

It loads Qwen2-VL with the class Qwen2VLForConditionalGeneration. I set torch.bfloat16 otherwise the model will be loaded with float32 parameters and will consume much more memory. If your GPU doesn't support bfloat16, you can replace it with torch.float16.

I use FlashAttention. If your GPU is not recent, replace "flash_attention_2" with "sdpa". SDPA is the PyTorch's implementation of scaled dot-product. Then, I set device_map="auto". "auto" means that the model will be split over several devices if your GPU doesn't have enough memory. To understand how device map works, check this article:

The Processor processes the text, image, and video.

The prompt can be formatted as follows, in JSON:

messages = [
    {
        "role": "user",
        "content": [
            {
                "type": "image",
                "image": "https://about.benjaminmarie.com/data/visual_samples/image0.jpg",
            },
            {"type": "text", "text": "How many dogs do you see? What are they doing?"},
        ],
    }
]

The “content” contains two items:

  • An image: Its type is “image” and it can be a URL pointing to an image or a local file. Here, I put the URL of the image above showing two dogs.

  • A text: This is what we want to ask the model. It refers to the image in this example.

Next, we need to process it, i.e., tokenize it and retrieve the image features:

text = processor.apply_chat_template(
    messages, tokenize=False, add_generation_prompt=True
)

image_inputs, video_inputs = process_vision_info(messages)
inputs = processor(
    text=[text],
    images=image_inputs,
    videos=video_inputs,
    padding=True,
    return_tensors="pt",
)

inputs = inputs.to("cuda")

We are in the “inference” mode so it’s very important to add “add_generation_prompt=True”. If you don’t, the tokenizer won’t append “\n<|im_start|>assistant\n” at the end of the prompt and the model will probably complete the user prompt rather than generating the “assistant” turn.

process_vision_info retrieves the visual features, then we pass them to the processor along with our text formatted by the chat template of the model.

Then, we make sure that the inputs are on the GPU. Note: I didn’t try inference with a CPU.

The final part is (almost) regular inference code with Transformers:

generated_ids = model.generate(**inputs, max_new_tokens=256)
generated_ids_trimmed = [
    out_ids[len(in_ids) :] for in_ids, out_ids in zip(inputs.input_ids, generated_ids)
]

output_text = processor.batch_decode(
    generated_ids_trimmed, skip_special_tokens=True, clean_up_tokenization_spaces=False
)

print(output_text)

“generated_ids” also contains the input token ids. “generated_ids_trimmed” only contains what has been generated by the model. The process decodes it, replaces ids with their corresponding tokens from the tokenizer vocabulary, and then detokenizes the output. “skip_special_tokens” doesn’t print the tokens used by the chat template.

For this example, Qwen2-VL is very good:

In the image, there are two dogs running on a dirt path. The dog on the left is a Corgi, and the one on the right is a Yorkie. They appear to be enjoying a walk together.

Run Qwen2-VL with Multiple Images

To run the model with several images, the code is the same. We only modify the inputs to include the images in the prompt. For this example, I chose two images showing cars:

messages = [
    {
        "role": "user",
        "content": [
            {
                "type": "image",
                "image": "https://about.benjaminmarie.com/data/visual_samples/image1.jpg",
            },
            {
                "type": "image",
                "image": "https://about.benjaminmarie.com/data/visual_samples/image2.jpg",
            },
            {"type": "text", "text": "Which car is better?"},
        ],
    }
]

Qwen2-VL generates:

The image shows two different types of cars: a classic Mercury sedan and a Tesla Model S.\n\n1. **Classic Mercury Sedan**:\n   - This is a classic Mercury sedan from the 1950s or 1960s.\n   - It has a traditional design with a rounded front and a boxy body.\n   - The car is parked on the side of a road, and there is a person standing next to it.\n\n2. **Tesla Model S**:\n   - This is a modern electric car designed by Tesla.\n   - It has a sleek, aerodynamic design with a long hood and a low profile.\n   - The car is parked in front of a concrete wall with the Tesla logo on it.\n\n**Comparison**:\n- **Design**: The classic Mercury sedan has a more traditional and boxy design, while the Tesla Model S is sleek and modern.\n- **Economy**: The Tesla Model S is an electric car, which means it does not produce emissions and is environmentally friendly.\n- **Range**: The Tesla Model S has a range of up to 480 miles on a single charge, making it a long-range vehicle.\n- **Technology**: The Tesla Model S is equipped with advanced technology features such as Autop

It’s not bad but not very straightforward. I didn’t ask for all these details, and due to the maximum sequence length that I set, “max_new_tokens=256”, I couldn’t get the answer to my prompt “Which car is better?”.

Run Qwen2-VL with a Video

We can provide a video to Qwen2-VL the same way we provided it with images. We only have to change the “type” to video. For the accepted format, I only confirmed that the MP4 format works.

I used a video of a traffic jam provided by this dataset:

  • shivi/video-demo

Download the video locally and then indicate its location in the JSON object:

messages = [
    {
        "role": "user",
        "content": [
            {
                "type": "video",
                "video": "/content/cars.mp4",
                "max_pixels": 360 * 420,
                "fps": 1.0,
            },
            {"type": "text", "text": "Describe this video."},
        ],
    }
]

Internally, the video is decomposed into frames and processed as multiple images by the model. Qwen2-VL doesn’t process the audio.

For this video, the model generates:

The image depicts a busy urban street scene with numerous vehicles, including cars and motorcycles, moving in both directions. The traffic appears to be congested, with some vehicles waiting at intersections or in slow-moving lanes. The street is flanked by tall buildings on either side, and there is a bridge or overpass visible in the background. The sky is overcast, suggesting it might be a cloudy day. The overall atmosphere is typical of a bustling city with heavy traffic.

This is a very good output for a 2B parameter model.

Conclusion

Qwen2-VL are very good and easy-to-use multimodal models. The 2B and 7B versions can run on consumer hardware, even though you may have to use the quantized version for Qwen2-VL 7B if you only have a 16 GB GPU.

A 72B version, probably based on Qwen2-72B, will also be released by Alibaba. I expect its performance to be outstanding for an open model. However, it won’t run on a single consumer GPU, even quantized. You will need at least 2*24 GB GPUs.
