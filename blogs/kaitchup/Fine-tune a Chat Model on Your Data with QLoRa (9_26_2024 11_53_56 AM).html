Your GPU is all you need

GPT-NeoX-20B is among the most popular open GPT models (Apache 2.0 License, commercial use allowed). You can fine-tune this model on an instruction dataset to obtain your own chatbot similar to ChatGPT.

But GPT-NeoX-20B is a 20-billion parameter model. It won’t directly fit on your GPU.

With quantization, e.g., making a 4-bit version of the model, and adapters, we can fine-tune and run it on consumer hardware.

For this, we can use QLoRa. If you are curious about QLoRa, you can find more information in one of my previous blog posts:

QLoRa: Fine-Tune a Large Language Model on Your GPU

·
May 30, 2023

Most large language models (LLM) are too big to be fine-tuned on consumer hardware. For instance, to fine-tune a 65 billion parameters model we need more than 780 Gb of GPU memory. This is equivalent to ten A100 80 Gb GPUs. In other words, you would need cloud computing to fine-tune your models.

Read full story

So we have a GPT-like model, how can we transform it into a chat model similar to ChatGPT?

For this, we need instruction datasets. This is how Vicuna, Alpaca, and Guacano models were trained: They are mainly models based on LLaMa and fine-tuned on instructions generated by OpenAI’s GPT-3. Since they are based on LLaMa, we can’t use them for commercial purposes. But you can if you fine-tune instead a GPT-NeoX model.

For instance, Alpaca and Guacano models were fine-tuned on these datasets:

  • Alpaca dataset: 52,000 instructions and demonstrations generated by GPT-3, in English

  • Open Assistant Guanaco dataset: 9,846 instructions extracted from the OpenAssistant conversation datasets. These are the highest-rated paths in the conversation tree.

In contrast to the Alpaca dataset, Guanaco is small but multilingual. Note: There is another instruction dataset called Guacano that is totally different. It is much larger and compiled from a different source of data. I don’t use it in this article since this dataset is not completely open, e.g., you need to provide personal information to get it.

In this article, I show how to fine-tune a 20 billion parameter model, GPT-NeoX-20b, on Alpaca, Guanaco, and your own datasets by using only your GPU. It also works on a free instance of Google Colab, in theory, but in practice, it would take so much time that you would have to restart the Google Colab runtime. I also discuss how you can create your own dataset of instructions in case Alpaca and Guanaco datasets don’t cover your tasks or languages of interest.

Set up your environment for fine-tuning with QLoRa

First, install all the following Python dependencies.

pip install -q -U bitsandbytes
pip install -q -U git+https://github.com/huggingface/transformers.git 
pip install -q -U git+https://github.com/huggingface/peft.git
pip install -q -U git+https://github.com/huggingface/accelerate.git
pip install -q -U datasets

Note: Hugging Face is updating Accelerate very frequently. Recently, they broke the support for QLoRa. If you have errors related to Accelerate, consider using an older version that you can obtain with “pip install accelerate=0.20.3”.

That’s all you need to use QLoRa and the datasets we have selected.

Prepare your datasets

First, we need the tokenizer of GPT-NeoX-20b:

from transformers import AutoTokenizer, AutoModelForCausalLM, BitsAndBytesConfig

model_name = "EleutherAI/gpt-neox-20b"
#Tokenizer
tokenizer = AutoTokenizer.from_pretrained(model_name)

#gpt-neox-20b doesn't have a PAD token
#We can use the EOS token for padding
tokenizer.pad_token = tokenizer.eos_token

We will need our own data collator for preparing the data. I prepared the following, inspired by the code released on QLoRa’s GitHub:

from datasets import load_dataset, Dataset
import torch, copy
import transformers
from typing import Optional, Dict, Sequence
from torch.nn.utils.rnn import pad_sequence
from dataclasses import dataclass, field

IGNORE_INDEX = -100
DEFAULT_PAD_TOKEN = "[PAD]"

@dataclass
class DataCollatorForCausalLM(object):
    tokenizer: transformers.PreTrainedTokenizer
    source_max_len: int
    target_max_len: int

    def __call__(self, instances: Sequence[Dict]) -> Dict[str, torch.Tensor]:
        sources = [f"{self.tokenizer.bos_token}{example['input']}" for example in instances]
        targets = [f"{example['output']}{self.tokenizer.eos_token}" for example in instances]
        # Tokenize
        tokenized_sources_with_prompt = self.tokenizer(
            sources,
            max_length=self.source_max_len,
            truncation=True,
            add_special_tokens=False,
        )
        tokenized_targets = self.tokenizer(
            targets,
            max_length=self.target_max_len,
            truncation=True,
            add_special_tokens=False,
        )
        # Build the input and labels for causal LM
        input_ids = []
        labels = []
        for tokenized_source, tokenized_target in zip(
            tokenized_sources_with_prompt['input_ids'],
            tokenized_targets['input_ids']
        ):
          input_ids.append(torch.tensor(tokenized_source + tokenized_target))
          labels.append(
              torch.tensor([IGNORE_INDEX for _ in range(len(tokenized_source))] + copy.deepcopy(tokenized_target))
          )


        # Apply padding
        input_ids = pad_sequence(input_ids, batch_first=True, padding_value=0)
        labels = pad_sequence(labels, batch_first=True, padding_value=IGNORE_INDEX)
        data_dict = {
            'input_ids': input_ids,
            'attention_mask':input_ids.ne(0),
        }

        if labels is not None:
            data_dict['labels'] = labels
        return data_dict

data_collator = DataCollatorForCausalLM(
    tokenizer=tokenizer,
    source_max_len=1024,
    target_max_len=256,
)

If you want to use Alpaca

We need to prepare a format for the data, as follows:

ALPACA_PROMPT_DICT = {
    "prompt_input": (
        "Below is an instruction that describes a task, paired with an input that provides further context. "
        "Write a response that appropriately completes the request.\n\n"
        "### Instruction:\n{instruction}\n\n### Input:\n{input}\n\n### Response: "
    ),
    "prompt_no_input": (
        "Below is an instruction that describes a task. "
        "Write a response that appropriately completes the request.\n\n"
        "### Instruction:\n{instruction}\n\n### Response: "
    ),
}

def extract_alpaca_dataset(example):
    if example.get("input", "") != "":
        prompt_format = ALPACA_PROMPT_DICT["prompt_input"]
    else:
        prompt_format = ALPACA_PROMPT_DICT["prompt_no_input"]
    return {'input': prompt_format.format(**example)}

Then, we load and format the data:


alpaca_data = load_dataset("tatsu-lab/alpaca")
alpaca_data = alpaca_data.map(extract_alpaca_dataset, remove_columns=['instruction'])

#We keep only the modified input and output columns in the dataset
alpaca_data = alpaca_data.remove_columns(
      [col for col in alpaca_data.column_names['train'] if col not in ['input', 'output']]
)
my_data = alpaca_data

If you want to use Guanaco

The format is more simple with fewer columns in the dataset.


guanaco_data = load_dataset("timdettmers/openassistant-guanaco")
guanaco_data = guanaco_data.map(lambda x: {
          'input': '',
          'output': x['text'],
      })
my_data = guanaco_data

If you want to use your own dataset

If you want to create your own chatbot specialized for some task, you can start by generating data with ChatGPT, or any other models that suits your task.

The Alpaca dataset was created that way and it has only cost $500 according to its creators.

Let’s say that you want to make a travel assistant.

You can generate instructions by generating prompts, e.g., “What is the best itinerary between [P1] and [P2] by [M].” where each tag could be replaced by the following values:

  • [P1] and [P2]= {Paris, Nice, Roma, Berlin, Madrid}

  • [M] = {bicycle, car, foot}

This can give you a lot of different combinations.

Once you have generated your dataset, you can revise it to correct the format or the errors made during the generation.

Next, you have to format it as an instruction dataset. Prepend the prompt with “### Human:” and then, immediately following the prompt, concatenate “### Assistant:” followed by the model output. Example:

### Human: What is the best itinerary between Paris and Nice by car?### Assistant: Take the highway[…]

Put all your instructions into a JSON object, one instruction per entry. You can use this format:

{'text': '### Human: What is the best itinerary between Paris and Nice by car?### Assistant: Take the highway[…]'}

For instance, if you saved all your instructions into a file “instruct.json”, one instruction per line, you only have to read the file as a dataset like this:

my_instructs = Dataset.from_json("instruct.json")
my_instructs = my_instructs.map(lambda x: {
          'input': '',
          'output': x['text'],
      })
my_data = my_instructs

Loading and quantizing GPT-NeoX

Note: In this section, I do the same as in my previous article on QLoRa.

We will use the following additional libraries:

from peft import prepare_model_for_kbit_training, LoraConfig, get_peft_model

Then we can load the model’s tokenizer and quantize the model:


#QLoRa configuration
quant_config = BitsAndBytesConfig(
    load_in_4bit=True,
    bnb_4bit_use_double_quant=True,
    bnb_4bit_quant_type="nf4",
    bnb_4bit_compute_dtype=torch.bfloat16
)

model = AutoModelForCausalLM.from_pretrained(model_name, quantization_config=quant_config, device_map={"":0}, use_cache=False)

I also activate gradient checkpointing to further reduce memory usage. If you have enough memory, you may skip this to speed up training.

model.gradient_checkpointing_enable()

Then, we have to convert the model to LoRa:

model = prepare_model_for_kbit_training(model)

config = LoraConfig(
    r=8, 
    lora_alpha=32, 
    target_modules=["query_key_value"], 
    lora_dropout=0.05, 
    bias="none", 
    task_type="CAUSAL_LM"
)

model = get_peft_model(model, config)

Finally, we can start training.

If you use a free instance of Google Colab, it should take around 20 hours with the Guanaco dataset, assuming that you save the checkpoints and restart the runtime at least once (Google will disconnect it anyway).

The following Trainer is only configured for testing that everything works.

trainer = transformers.Trainer(
    model=model,
    train_dataset=my_data['train'],
    data_collator=data_collator,
    args=transformers.TrainingArguments(
        per_device_train_batch_size=1,
        gradient_accumulation_steps=4,
        warmup_steps=2,
        max_steps=10,
        learning_rate=2e-4,
        fp16=True,
        logging_steps=1,
        output_dir="outputs",
        optim="paged_adamw_8bit",
        remove_unused_columns=False

    )
)
trainer.train()

Once you have confirmed that it is running, you can increase max_steps (eg. 3000), warmup_steps (eg. 100), and logging steps (eg. 50).

Note that if you don’t put “remove_unused_columns” to False, the Trainer will remove columns from the dataset that are necessary, before collating the data. This will trigger an error.

If you have a bigger machine, consider increasing “per_device_train_batch_size”.

To test inference, you can run the following:

text = "### Human: Tell me about gravity.### Assistant:"
device = "cuda:0"

inputs = tokenizer(text, return_tensors="pt").to(device)
outputs = model.generate(**inputs, max_new_tokens=10) 
print(tokenizer.decode(outputs[0], skip_special_tokens=True))

To allow longer output, you must increase “max_new_tokens”. This also significantly increases inference time.

Conclusion

Thanks to quantization, open LLMs, and open datasets, you can now train your own chatbot for free.

You can even do it for commercial purposes since the datasets and models are all distributed with an Apache 2.0 license. Nonetheless, note that Alpaca and Guacano were generated with OpenAI models. There may be a conflict with OpenAI’s terms of use as I described here:
