Multimodal RAG with ColPali and Qwen2-VL on Your Computer

Retrieve and exploit information from PDFs without OCR

A standard Retrieval-Augmented Generation (RAG) system retrieves chunks of text from documents and incorporates them into the context of a user’s prompt. By leveraging in-context learning, a large language model (LLM) can use the retrieved information to provide more relevant and accurate answers.

However, what if the documents contain a lot of tables, plots, and images?

A standard RAG system will often overlook this additional information. When processing text, such as from PDF files, traditional RAG systems rely on OCR tools. Unfortunately, OCR accuracy is typically low for documents with complex formatting, tables, or images. As a result, irrelevant or misformatted text chunks may be indexed, which, when retrieved and injected into the LLM’s context, can negatively impact the quality of its answers.

To avoid relying on OCR and to harness additional information from images, plots, and tables, we can implement a multimodal RAG system. By using a vision-language model (VLM) like Qwen2-VL or Pixtral, we can build a RAG system capable of reading both text and visual data.

In this article, I explain how a multimodal RAG system works. We’ll then see how to set up a multimodal RAG system using ColPali to chunk and encode images and PDF documents, while Qwen2-VL is used to process and exploit the retrieved multimodal information to answer user prompts. For instance, I applied this approach to a complex scientific paper from the arXiv’s Earth and Planetary Astrophysics section.

Multimodal RAG with a Visual Retriever

For text, a RAG system will encode the user prompt with text embeddings and compare the results with the encoded chunks of documents indexed in a database.

However, for a multimodal RAG, the indexed chunks of documents are not text. We can’t use simple text embeddings for them and the user prompt. We need embeddings that can encode both visual and language features. We need a visual retriever.

There are only a few good visual retrievers. ColPali is one of them.

ColPali is designed for efficient document retrieval, combining visual and textual data. It extends PaliGemma-3B and generates ColBERT-style multi-vector representations of text and images. This enables the model to index documents by analyzing their visual features. The model was introduced in this paper:

ColPali: Efficient Document Retrieval with Vision Language Models

A key innovation of ColPali is mapping image patches into a similar latent space as text. It unlocks efficient interaction between text and images using the ColBERT strategy. The training dataset included 127,460 query-page pairs from academic datasets (63%) and web-crawled PDF documents (37%). These documents were augmented with VLM-generated pseudo-questions.

How to Use ColPali for RAG

With ColPali, we can encode and index both text and visual features from documents. This includes tables, plots, and images. It can be applied to PDF files.

When the user sends a prompt, it is encoded with ColPali, and the resulting embeddings are used to retrieve similar chunks of documents from the database that may contain text and images.

The RAG system then injects what was retrieved into the user prompt. Such multimodal information can’t be exploited by a standard LLM. We need to use a chat model based on a VLM, for instance, Florence-2, Phi-3 vision, Pixtral, or Qwen2-VL are all good VLMs for RAG.

Building a Multimodal RAG System with ColPali and Qwen2-VL

Note: The code below largely borrows pieces of notebooks made by Answer.AI and merveenoyan’s smol-vision repository. Check their work! I especially recommend the Smol-vision repository which proposes several cool notebooks showcasing VLMs.

Setting Up your Environment for ColPali and Qwen2-VL

First, install the following dependencies:

pip install --upgrade git+https://github.com/huggingface/transformers.git byaldi accelerate flash-attn qwen_vl_utils pdf2image
!sudo apt-get install -y poppler-utils

  • I needed to install Transformers from source for running Qwen2-VL.

  • FlashAttention (flash_attn) isn’t mandatory but without it, you won’t be able to run Qwen2-VL 2B on a consumer GPU (24 GB).

  • Byaldi makes very easy the use of ColPali for RAG. It’s a wrapper of the ColPali repository.

  • qwen_vl_utils is made by the Qwen team to facilitate the processing of the input for Qwen VL.

  • pdf2image is used to convert the PDF pages into images. This is necessary since Qwen2-VL can’t encode PDF files.

  • poppler-utils is required by Byaldi to index the PDF pages.

To test my multimodal RAG system, I used this document:

Slow Rotation for the Super-Puff Planet Kepler-51d

This is a PDF of 10 pages with mainly text and a few images.

I’ve downloaded it, named it "1.pdf" and put it in the directory "documents/". You can put several PDF files and images in this directory. They will all be indexed by Byaldi.

Then, import the following in your Python script:

from byaldi import RAGMultiModalModel
from transformers import Qwen2VLForConditionalGeneration, AutoTokenizer, AutoProcessor
from qwen_vl_utils import process_vision_info
import torch
from pdf2image import convert_from_path

Load the Models for Multimodal RAG

RAG = RAGMultiModalModel.from_pretrained("vidore/colpali")
model = Qwen2VLForConditionalGeneration.from_pretrained("Qwen/Qwen2-VL-7B-Instruct", torch_dtype=torch.bfloat16,  attn_implementation="flash_attention_2", device_map="cuda")

We load ColPali with RAGMultiModalModel from byaldi. byaldi is a very young project and doesn’t have many options yet. We can’t quantize ColPali with bitsandbytes and not even decide whether we want to load with float16 or bfloat16.

For the multimodal chat, I use Qwen/Qwen2-VL-7B-Instruct. It’s currently my favorite VLM. You could also use Florence-2 or Phi-3 Vision but I think these models are much less accurate to encode images with a lot of text.

Note that using Qwen/Qwen2-VL-7B-Instruct requires at least a 32 GB GPU. You can use the quantize version or the 2B version instead if you only have a consumer GPU. However, I would expect these versions to be much less accurate for tasks involving a lot of text in images.

Moreover, I really recommend a GPU supporting FlashAttention, i.e., an Ampere or a more recent GPU. Otherwise, it will consume much more memory.

RAG.index(

    overwrite=True # Whether to overwrite an index if it already exists. If False, it'll return None and do nothing if `index_root/index_name` exists.

Query a Multimodal RAG with Byaldi

The user query that we will test:

text_query = "What is the type of the star hosting the Kepler-51 planetary system?"

The answer to this question is in the first page of the document:

ColPali will encode this text and use the embeddings to retrieve the relevant pages from the indexed document.

results = RAG.search(text_query, k=1)
print(results)

Here, we retrieve the top 1 document (k=1). k is a hyperparameter. If you retrieve more documents, inference will be more costly as the prompt sent to Qwen2-VL will contain more pages. It will consume more memory. We may also introduce irrelevant documents.

It outputs:

[{'doc_id': 0, 'page_num': 1, 'score': 23.875, 'metadata': {}, 'base64': None}]

It is a good start. The first page, which contains the answer to our question, is retrieved.

Prompting Qwen2-VL in a Multimodal RAG System

Remember that Qwen2-VL is a VLM. It can’t deal with PDFs. We have to convert each page into an image.

images = convert_from_path("documents/1.pdf")

Then, using the results returned by Byaldi for our query, we can get the corresponding image:

image_index = results[0]["page_num"] - 1

Then, the next step is a standard inference pipeline for Qwen2-VL:

processor = AutoProcessor.from_pretrained("Qwen/Qwen2-VL-7B-Instruct", trust_remote_code=True)


messages = [
    {
        "role": "user",
        "content": [
            {
                "type": "image",
                "image": images[image_index],
            },
            {"type": "text", "text": text_query},
        ],
    }
]

text = processor.apply_chat_template(
    messages, tokenize=False, add_generation_prompt=True
)

image_inputs, video_inputs = process_vision_info(messages)
inputs = processor(
    text=[text],
    images=image_inputs,
    videos=video_inputs,
    padding=True,
    return_tensors="pt",
)
inputs = inputs.to("cuda")


generated_ids = model.generate(**inputs, max_new_tokens=50)
generated_ids_trimmed = [
    out_ids[len(in_ids) :] for in_ids, out_ids in zip(inputs.input_ids, generated_ids)
]
output_text = processor.batch_decode(
    generated_ids_trimmed, skip_special_tokens=True, clean_up_tokenization_spaces=False
)
print(output_text)

Conclusion

Implementing a multimodal RAG system with ColPali and Qwen2-VL significantly enhances the retrieval and use of both textual and visual information from complex documents like PDFs containing tables, plots, and images.

By moving beyond the limitations of OCR and text embeddings, this approach employs a visual retriever to index visual features and a VLM to process and understand user queries. This results in more accurate and relevant responses, making it a powerful solution for applications that require a comprehensive understanding of multimodal content.

For a more memory-efficient multimodal RAG system, you could try Florence-2. However, don’t expect it to be accurate for large PDFs. I would rather use it for applications relying on a database made of standard images.
