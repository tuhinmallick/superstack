Instruct large language models (LLMs) are specialized versions of base LLMs that have been fine-tuned on instruction datasets. These datasets consist of pairs of instructions or questions and corresponding answers, which are either written by humans or generated by AI. Instruct LLMs are commonly used in chat applications, with GPT-4 being a prominent example.

Instruct LLMs undergo several post-training stages, including supervised fine-tuning, reinforcement learning with human feedback (RLHF), and Direct Preference Optimization (DPO). Through these processes, the models are trained to respond more effectively to human prompts by adhering to a specific format or chat template, often defined by special tokens added to their vocabulary.

As a result, instruct LLMs produce outputs formatted according to the templates they learned during fine-tuning, whereas base LLMs generate text by predicting the next token without these learned constraints.

When we want to fine-tune an LLM on our own data, these differences raise an important question: Should we fine-tune the base version or the instruct version?

How Are Instruct LLMs Different From Base LLMs?

It is now common for LLM makers to release two versions of their LLMs at the same time: the base version and an instruct version. The instruct version is also sometimes called “chat” version. For instance, for Llama 3, Gemma 2, and Qwen2, we have:

  • meta-llama/Meta-Llama-3.1-8B and meta-llama/Meta-Llama-3.1-8B-Instruct

  • google/gemma-2-9b and google/gemma-2-9b-it

  • Qwen/Qwen2-7B and Qwen/Qwen2-7B-Instruct

To understand why we shouldn’t fine-tune the instruct version, we must first have a look at how they were trained and how we use them.

Base LLMs

LLMs are pre-trained on trillions of tokens. During this phase, the training objective is the next-token prediction. In other words, after pre-training, LLMs are only capable of generating the most probable tokens following a prompt provided by the user. They are not restricted and won’t follow a template.

Let’s try Llama 3.1 8B with the following prompt:

Give me a list of 20 countries near Indonesia and tell me which is the largest.

We can generate the output of Llama 3.1 8B for this prompt with:

import transformers
import torch

model_id = "meta-llama/Meta-Llama-3.1-8B"

pipeline = transformers.pipeline(
    "text-generation", model=model_id, model_kwargs={"torch_dtype": torch.bfloat16}, device_map="auto"
)

outputs = pipeline("Give me a list of 20 countries near Indonesia and tell me which is the largest.", max_new_tokens=256)
print(outputs[0])

The model didn’t directly answer our prompt. It only continues, or completes, the text that we provided as a prompt. A base model can be very creative this way but may not answer what we would have expected. Moreover, it can be unsafe as it may generate inappropriate answers.

Another specificity of base LLMs is that they are not trained to stop generating. The output will always be truncated once the maximum sequence length, defined by the model or as an inference hyperparameter, is reached. As the context gets longer during generation, the model will tend to repeat itself and generate content unrelated to the prompt provided by the user.

To be useful, base LLMs must be fine-tuned.

Instruct LLMs

Instruct LLMs are base LLMs but fine-tuned with a complex pipeline to teach them how to answer user instructions and to be aligned with human preferences. They also underwent, in most cases, training to make their responses safer for users and avoid sensitive topics.

For instance, what Meta did to turn Llama 3.1 into Llama 3.1 Instruct is a complex recipe. It is described in Section 4 of the paper presenting Llama 3.1:

The Llama 3 Herd of Models

In a nutshell, they performed a supervised fine-tuning (SFT) step to teach the model how to answer user instructions and then ran a DPO training to align the model with human preferences, using the DPO loss in combination with the negative log-likelihood. They ran SFT and DPO training with various hyperparameters and then averaged the models.

This is a complex template for multi-turn dialogues that can be augmented with tool calls. This template is automatically applied by most inference frameworks supporting Llama 3.1. We can also see that the template contains system instructions, such as “Given the following functions, please respond with a JSON for a function call”. Instruct LLMs are fine-tuned to follow exactly these system instructions.

With Hugging Face Transformers, the prompt is passed to the model as a JSON object, then the chat template is automatically applied to this object.

import transformers
import torch

model_id = "meta-llama/Meta-Llama-3.1-8B-Instruct"

pipeline = transformers.pipeline(
    "text-generation",
    model=model_id,
    model_kwargs={"torch_dtype": torch.bfloat16},
    device_map="auto",
)

messages = [
    {"role": "user", "content": "Give me a list of 20 countries near Indonesia and tell me which is the largest."},
]

outputs = pipeline(
    messages,
    max_new_tokens=256,
)
print(outputs[0])

Output:

I'm not able to translate the sentence into Aymara.

This example is interesting because it shows that the model has been taught that it doesn't know Aymara (a language from the Bolivian Andes). If you want to fine-tune this instruct model to translate this language, you may have a hard time teaching it that it now knows this language. The base version would learn faster.

We can clearly see in these two examples the impact of the post-training performed by Meta to make Llama 3.1 8B able to answer instructions. Note also that the model stopped generating once it answered the prompt.

Why We Shouldn’t Fine-tune an instruct LLM?

At that point, you may have an intuition on why instruct LLMs shouldn’t be fine-tuned.

Fine-tuning updates the weights of the model given the fine-tuning data. At each fine-tuning step, the model moves away from what it has previously learned. However, unless the model strongly overfits the fine-tuning data, it will not completely forget what was learned.

In practice, it means that if we fine-tune the instruct LLMs, we are undoing what Meta did (for Llama 3) with SFT and DPO. The model will lose some of its capacities to generate answers aligned with human preferences. To entirely preserve what Meta did, we would have to apply the exact same process used by Meta for SFT and DPO, using our new fine-tuning data. However, since we don’t have enough details about what Meta did, this is not possible.

Moreover, since the model doesn’t completely forget what it has learned during SFT and DPO, some of the safety guidelines and biases introduced by Meta will remain while they could be undesirable for our new fine-tuning.

Remember also that instruct LLMs are fine-tuned with their own chat template and their own system instructions. Fine-tuning an instruct LLMs will introduce a conflict between the new system instructions (if any) and the system instructions previously learned by the model. The same will happen with the template. The model might mix old and new templates within the same answer.

In other words, fine-tuning an instruct LLM yields unexpected results.

Conclusion

In most cases, fine-tuning a base LLM will yield better results than fine-tuning the instruct version because it is not biased by the arbitrary post-training recipe applied by its maker. The base LLM will learn your fine-tuning data's characteristics (domain, style, etc.) faster.

In theory, fine-tuning the instruct LLMs for a chat application should work but may yield unexpected results: conflict of chat template, system instruction leakage, denial to answer instructions due to the safety training of the instruct LLMs, etc. That being said if you use the same chat template learned by the instruct LLMs, it should be fine but I don’t recommend it.

In the notebook, you will find inference code examples using adapters for Llama 3.1 and Llama 3.1 Instruct trained on the same dataset. In my opinion, the adapter fine-tuned on the base version of Llama 3.1 performs better but the differences are not always obvious.

For other types of applications, e.g., translation, paraphrasing, or summarization, fine-tuning the base LLMs will certainly yield better results than fine-tuning the instruct version. For these applications, we don’t need a chat template and we don’t need the LLM to be aligned with human preferences or trained to be safe. For instance, the instruct version of Llama 3.1 or Gemma still refuses to translate some languages because they were taught during post-training that they don’t support them. Undoing this training to introduce new knowledge or capacities is feasible but difficult.

Fine-tuning the base LLMs is a better alternative.
