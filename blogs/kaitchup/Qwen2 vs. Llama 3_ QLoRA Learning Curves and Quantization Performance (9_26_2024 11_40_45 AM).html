Qwen2 is a new family of LLMs by Alibaba Cloud, including pre-trained and instruction-tuned models with 5 sizes available: 0.5B, 1.5B, 7B, 57B-A14B (mixture of experts), and 72B.

According to their own evaluation, Qwen2 significantly outperforms Llama 3 in most tasks. Qwen2 looks like a good alternative to Llama 3 and is very suitable for consumer hardware thanks to the availability of small models such as the 0.5B and 1.5B versions.

Are the Qwen2 LLMs still better than Llama 3 after quantization? Do they learn faster and better than Llama 3 with QLoRA fine-tuning?

In this article, I review Qwen2 and answer these questions. We will see the main differences in architectures between Qwen2 and Llama 3. I tested the robustness of Qwen2 to quantization and compared its performance, once quantized, with Llama 3. Finally, we will compare the learning curves of Llama 3 and Qwen2 obtained with QLoRA fine-tuning. Overall, Qwen2 seems more robust to quantization and a good alternative to Llama 3.

Qwen2 Architecture

We don’t have much information about Qwen2 for now. They will release a technical report “very soon”. For now, we only know what they revealed in their blog post:

Hello Qwen2

All the models use Grouped-Query Attention (GQA) to speed up inference and reduce the size of the KV cache. Qwen2-7B and Qwen2-72B support a context length of 128k tokens thanks to YaRN. Note: The number “#Params” for Qwen2-7B seems wrong in the table below. The model released on Hugging Face has 7.62B parameters.

Qwen2 officially supports 28 languages:

It makes it superior to other LLMs in most multilingual tasks.

For the instruct fine-tuning, they used “a combination of supervised fine-tuning, reward model training, and online DPO training”. We don’t know more for now.

The performance of Qwen2 looks impressive on public benchmarks.

However, as usual, since we don’t know the training data and the evaluation details, my recommendation is to ignore these scores and evaluate the models by yourselves for your own applications.

The Qwen2 models use the Llama architecture. For instance, the architecture of Qwen2-7B:

Qwen2ForCausalLM(
  (model): Qwen2Model(
    (embed_tokens): Embedding(152064, 3584)
    (layers): ModuleList(
      (0-27): 28 x Qwen2DecoderLayer(
        (self_attn): Qwen2FlashAttention2(
          (q_proj): Linear4bit(in_features=3584, out_features=3584, bias=True)
          (k_proj): Linear4bit(in_features=3584, out_features=512, bias=True)
          (v_proj): Linear4bit(in_features=3584, out_features=512, bias=True)
          (o_proj): Linear4bit(in_features=3584, out_features=3584, bias=False)
          (rotary_emb): Qwen2RotaryEmbedding()
        )
        (mlp): Qwen2MLP(
          (gate_proj): Linear4bit(in_features=3584, out_features=18944, bias=False)
          (up_proj): Linear4bit(in_features=3584, out_features=18944, bias=False)
          (down_proj): Linear4bit(in_features=18944, out_features=3584, bias=False)
          (act_fn): SiLU()
        )
        (input_layernorm): Qwen2RMSNorm()
        (post_attention_layernorm): Qwen2RMSNorm()
      )
    )
    (norm): Qwen2RMSNorm()
  )
  (lm_head): Linear(in_features=3584, out_features=152064, bias=False)
)

Qwen2 has a larger vocabulary: 152,064 entries against 128,256 for Llama 3. However, Llama 3 has larger embeddings, 4096 dimensions, while Qwen2 has 3584 dimensions. In other words, Qwen2 has token embeddings with 545M parameters against 525M parameters for Llama 3. The language modeling head also has 545M parameters. This is a total of 1.1B parameters for the embeddings, i.e., 2.2 GB for 16-bit parameters.

For running the models on consumer hardware, this is important information to take into account. These parameters are not quantized by most quantization methods. The 2.2 GB occupied by the embeddings won’t be compressed.

On the other hand, Qwen2-7B has fewer non-embedding parameters than Llama 3 8B. While Qwen2-7B has a higher intermediate size, 18,944 against 14,336 for Llama 3 8B, it has fewer layers, 28 layers against 32 for Llama 3 8B.

It yields 7.62B parameters, i.e., 430M parameters less than Llama 3 8B.

Fine-tuning Qwen2 with QLoRA

Since Qwen2 uses the Llama architecture, fine-tuning Qwen2 LLMs is straightforward. We can use the same code that we used to fine-tune Llama 3 with only a few, but important, modifications.

First, to avoid endless generation, we have to check how the EOS token is handled by Qwen2. I explained how to do this in the article:

  • I found that, like Llama 3, Qwen2’s tokenizer doesn’t support “add_eos_token”. We have to add the EOS token manually to all the training examples.

  • The EOS token is pre-trained. We don’t need to fully fine-tune the token embeddings and language modeling head to train the EOS token (but you might still want to do it if your data belongs to a very specific domain/style).

  • Qwen2 doesn’t have a pad token. As we saw in the previous article cited above, don’t use the EOS token for padding. Instead, Qwen2 base LLMs have other special tokens such as <im_end> that we can use for padding.

Except for these modifications, the fine-tuning code is the same as for Llama 3 (I put in bold the lines specific to Qwen2):

import torch, os, multiprocessing
from datasets import load_dataset
from peft import LoraConfig, prepare_model_for_kbit_training
from transformers import (
    AutoModelForCausalLM,
    AutoTokenizer,
    BitsAndBytesConfig,
    set_seed
)
from trl import SFTTrainer, SFTConfig

set_seed(1234)

#use bf16 and FlashAttention if supported
if torch.cuda.is_bf16_supported():
  os.system('pip install flash_attn')
  compute_dtype = torch.bfloat16
  attn_implementation = 'flash_attention_2'
else:
  compute_dtype = torch.float16
  attn_implementation = 'sdpa'

model_name = "Qwen/Qwen2-7B"
#Tokenizer
tokenizer = AutoTokenizer.from_pretrained(model_name, use_fast=True)
tokenizer.pad_token = "<|im_end|>"
tokenizer.pad_token_id = 151645
tokenizer.padding_side = 'left'

ds = load_dataset("timdettmers/openassistant-guanaco")

#Add the EOS token
def process(row):
    row["text"] = row["text"]+tokenizer.eos_token
    return row

ds = ds.map(
    process,
    num_proc= multiprocessing.cpu_count(),
    load_from_cache_file=False,
)

print(ds['train']['text'][0])

bnb_config = BitsAndBytesConfig(
        load_in_4bit=True,
        bnb_4bit_quant_type="nf4",
        bnb_4bit_compute_dtype=compute_dtype,
        bnb_4bit_use_double_quant=True,
)
model = AutoModelForCausalLM.from_pretrained(
          model_name, quantization_config=bnb_config, device_map={"": 0}, attn_implementation=attn_implementation
)

print(model)

model = prepare_model_for_kbit_training(model, gradient_checkpointing_kwargs={'use_reentrant':True})
model.config.use_cache = False # Gradient checkpointing is used by default but not compatible with caching

peft_config = LoraConfig(
        lora_alpha=16,
        lora_dropout=0.05,
        r=16,
        bias="none",
        task_type="CAUSAL_LM",
        target_modules= ['k_proj', 'q_proj', 'v_proj', 'o_proj', "gate_proj", "down_proj", "up_proj"]
)

training_arguments = SFTConfig(
        output_dir="./Qwen2_7B_QLoRA",
        eval_strategy="steps",
        do_eval=True,
        optim="paged_adamw_8bit",
        per_device_train_batch_size=8,
        gradient_accumulation_steps=4,
        per_device_eval_batch_size=8,
        log_level="debug",
        save_strategy="epoch",
        logging_steps=25,
        learning_rate=1e-4,
        fp16 = not torch.cuda.is_bf16_supported(),
        bf16 = torch.cuda.is_bf16_supported(),
        eval_steps=25,
        num_train_epochs=1,
        warmup_ratio=0.1,
        lr_scheduler_type="linear",
        dataset_text_field="text",
        max_seq_length=512,
)

trainer = SFTTrainer(
        model=model,
        train_dataset=ds['train'],
        eval_dataset=ds['test'],
        peft_config=peft_config,
        tokenizer=tokenizer,
        args=training_arguments,
)

trainer.train()

I fine-tuned Llama 3 8B and Qwen2 7B with this code, using the same hyperparameters on the same dataset to compare their learning curves:

Qwen2 7B learns much faster than Llama 3, yielding a lower loss while requiring as much memory as Llama 3 8B. Of course, it’s only one experiment with one dataset and one set of hyperparameters. It’s far from enough to conclude that QLoRA fine-tuning works better with Qwen2 7B than Llama 3 8B. Nonetheless, it shows that Qwen2 7B can be a good alternative to Llama 3 8B.

Is Qwen2 More Robust to Quantization than Llama 3?

In a previous article, we saw that Llama 3 is difficult to accurately quantize, especially with GPTQ which is one of the most popular quantization algorithms. 4-bit quantization with GPTQ degrades Llama 3 8B’s accuracy down to the same level as Llama 2 7B. We saw that bitsandbytes quantization is much more accurate for Llama 3 8B.

Following the same methodology, I quantized Qwen2 0.5B, 1.5B, and 7B, with GPTQ (8, 4, 3, and 2 bits) and bitsandbytes (8 and 4 bits) and compared the accuracy of the models on three zero-shot tasks: Winogrande, HellaSwag, and Arc Challenge:

The main observations:

  • Qwen2 is very robust to 4-bit quantization with an accuracy that doesn’t drop as much as the accuracy of Llama 3 8B with 4-bit GPTQ. This observation also applies to the smallest model which is surprising as small LLMs are notoriously difficult to quantize.

  • For Qwen2, GPTQ seems to perform slightly better, on average, than bitsandbytes.

  • Llama 3 8B (not quantized) has a better accuracy than Qwen2 7B but once quantized to 4-bit with GPTQ Qwen2 7B is better.

We can conclude from these observations that Qwen2 is very robust to quantization. If you need to quantize your LLM to 4-bit while keeping it fast for inference, I recommend GPTQ. In this situation, Qwen2 7B seems a better option than Llama 3 8B.

As for its memory consumption, 4-bit Qwen2 7B, which occupies 5.58 GB, is slightly smaller than the 5.74 GB of 4-bit Llama 3 8B.

Conclusion

Qwen2 is very robust to quantization. We saw that for QLoRA fine-tuning and GPTQ quantization, Qwen2 7B is a good alternative to Llama 3 8B. Moreover, according to Alibaba’s evaluation, Qwen2 is better than Llama 3 in most tasks.

Another main advantage of Qwen2 over Llama 3 is its support of many more languages. If your application needs a small and fast LLM or a multilingual LLM, you should consider using Qwen2 7B.
